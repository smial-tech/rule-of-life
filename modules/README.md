# Modules folder

This folder contains the codebase for the project's modules or features. Each module is a self-contained codebase (API and UI) that can be developed and tested independently. The modules are organized into separate folders based on their functionality or purpose.

Each module should have the following structure:

- api/ (API codebase: repositories, services, routes, models, etc.)
- components/ (UI components)
- lib/ (common module's utility functions and helpers)
- hooks/ (common module's hooks)
